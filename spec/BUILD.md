# OMP Build Specification

## 1. Overview

OMP is built with TypeScript and bundled with esbuild. Three separate bundles are produced: hook bundles, MCP server, and CLI entry point. All source is in `src/`; output goes to `dist/`.

## 2. Directory Structure

```
src/
  agents/          ← 18 agent modules
  hooks/           ← 6 hook modules
  skills/          ← 30+ skill modules
  cli/             ← CLI entry point
  mcp/             ← MCP server
  core/            ← shared interfaces, HudState, AgentParams, etc.
dist/              ← generated by build
  agents/
  hooks/           ← hook bundles
  skills/
  cli/
  mcp/server.js    ← MCP server bundle
scripts/           ← build helper scripts
```

## 3. esbuild Configuration

Three separate esbuild configurations produce the three bundles.

### 3.1 Hooks Bundle

Target: `dist/hooks/bundle.js`

```javascript
// build/hooks.mjs
import * as esbuild from 'esbuild';

await esbuild.build({
  entryPoints: [
    'src/hooks/keyword-detector.ts',
    'src/hooks/delegation-enforcer.ts',
    'src/hooks/model-router.ts',
    'src/hooks/token-tracker.ts',
    'src/hooks/hud-emitter.ts',
    'src/hooks/stop-continuation.ts',
  ],
  bundle: true,
  platform: 'node',
  target: 'node18',
  format: 'esm',
  outdir: 'dist/hooks',
  split: false,        // single bundle for fast loading
  minify: true,
  sourcemap: true,
  external: [],
  define: {
    'process.env.NODE_ENV': '"production"',
  },
  banner: {
    js: '#!/usr/bin/env node',
  },
});
```

### 3.2 MCP Server Bundle

Target: `dist/mcp/server.js`

```javascript
// build/mcp.mjs
import * as esbuild from 'esbuild';

await esbuild.build({
  entryPoints: ['src/mcp/server.ts'],
  bundle: true,
  platform: 'node',
  target: 'node18',
  format: 'cjs',        // CJS for MCP stdio compatibility
  outfile: 'dist/mcp/server.js',
  minify: true,
  sourcemap: true,
  external: ['better-sqlite3'],
  define: {
    'process.env.NODE_ENV': '"production"',
  },
});
```

### 3.3 CLI Bundle

Target: `dist/cli/index.js`

```javascript
// build/cli.mjs
import * as esbuild from 'esbuild';

await esbuild.build({
  entryPoints: ['src/cli/index.ts'],
  bundle: true,
  platform: 'node',
  target: 'node18',
  format: 'cjs',
  outfile: 'dist/cli/index.js',
  minify: false,        // CLI readable for debugging
  sourcemap: true,
  external: [],
  define: {
    'process.env.NODE_ENV': '"production"',
  },
  banner: {
    js: '#!/usr/bin/env node',
  },
});
```

## 4. npm Scripts

```json
{
  "scripts": {
    "build": "npm run build:hooks && npm run build:mcp && npm run build:cli",
    "build:hooks": "node build/hooks.mjs",
    "build:mcp": "node build/mcp.mjs",
    "build:cli": "node build/cli.mjs",
    "build:all": "tsc --noEmit && npm run build",
    "dev": "tsc --watch",
    "test": "node --test tests/**/*.test.ts",
    "lint": "eslint src --ext .ts",
    "typecheck": "tsc --noEmit"
  }
}
```

## 5. External Dependencies

The following packages are external (not bundled):

| Package | Used by | Reason |
|---------|---------|--------|
| `better-sqlite3` | `src/core/state.ts` | Native SQLite bindings; must not be bundled |
| `@anthropic-ai/copilot-cli` | `src/cli/index.ts`, `src/mcp/server.ts` | Peer dep; resolved at runtime |
| `esbuild` | build scripts only | Dev dependency; not in runtime bundle |

## 7. Build Output Verification

After each build:
1. Verify `dist/hooks/bundle.js` contains all 6 hook exports
2. Verify `dist/mcp/server.js` is non-empty and has no TypeScript runtime errors
3. Verify `dist/cli/index.js` is executable (`chmod +x`)
4. Run `npm run typecheck` to confirm no type errors
