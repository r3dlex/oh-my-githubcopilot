# OMP Build Specification

## 1. Overview

OMP is built with TypeScript and bundled with esbuild. A single `esbuild.config.mts` at the project root produces all output. All source is TypeScript (`.mts`); output is ESM (`.mjs`). There is no TypeScript compilation step in the build pipeline — esbuild handles TypeScript directly via the `tsx` loader.

## 2. Directory Structure

```
src/
  hooks/             ← 6 hook source files (.mts)
  mcp/               ← MCP server source (.mts)
  core/              ← shared types, interfaces (PSM, HUD, etc.)
  cli/               ← (future) CLI source; currently src/index.mts
  index.mts          ← CLI entry point
dist/                ← generated by build
  hooks/             ← 6 individual hook bundles (.mjs)
  mcp/server.mjs     ← MCP server bundle
  skills/            ← skill bundles (.mjs) — not yet populated
bin/
  omp.mjs             ← CLI bundle (executable)
scripts/              ← (unused — no helper scripts)
build/                ← (unused — was planned for separate configs)
```

**What does NOT exist:**
- `src/agents/` — agents are Claude Code subagents, not local bundles
- `build/hooks.mjs`, `build/mcp.mjs`, `build/cli.mjs` — all esbuild config is in `esbuild.config.mts`
- `dist/agents/` — no agent bundles
- `dist/cli/` — CLI goes to `bin/omp.mjs`, not `dist/cli/`
- `dist/mcp/server.js` — output is `.mjs`, not `.js`

## 3. esbuild Configuration

`esbuild.config.mts` at project root is the single build definition. It runs via `node --import tsx esbuild.config.mts`.

### 3.1 Hooks — Individual Bundles

Six hooks are built as independent bundles, each a standalone ESM module. esbuild is invoked in a loop.

```typescript
import { build } from "esbuild";

const hookEntries = [
  "src/hooks/keyword-detector.mts",
  "src/hooks/delegation-enforcer.mts",
  "src/hooks/stop-continuation.mts",
  "src/hooks/token-tracker.mts",
  "src/hooks/model-router.mts",
  "src/hooks/hud-emitter.mts",
];

for (const entry of hookEntries) {
  const name = entry.replace("src/hooks/", "").replace(".mts", "");
  await build({
    entryPoints: [entry],
    bundle: true,
    platform: "node",
    target: "node22",
    format: "esm",
    outdir: "dist/hooks",
    outExtension: { ".js": ".mjs" },
    external: ["better-sqlite3"],
    sourcemap: true,
    minify: false,
  });
  console.log(`Built hook: dist/hooks/${name}.mjs`);
}
```

### 3.2 MCP Server — Single Bundle

```typescript
await build({
  entryPoints: ["src/mcp/server.mts"],
  bundle: true,
  platform: "node",
  target: "node22",
  format: "esm",
  outdir: "dist/mcp",
  outExtension: { ".js": ".mjs" },
  external: ["better-sqlite3"],
  sourcemap: true,
  minify: false,
});
// Output: dist/mcp/server.mjs
```

### 3.3 CLI Tool — Single Bundle

```typescript
await build({
  entryPoints: ["src/index.mts"],
  bundle: true,
  platform: "node",
  target: "node22",
  format: "esm",
  outfile: "bin/omp.mjs",
  external: ["better-sqlite3"],
  sourcemap: true,
  minify: false,
});
// Output: bin/omp.mjs (referenced as bin.omp in package.json bin field)
```

### 3.4 Skills — Individual Bundles

Two skills are currently built as individual ESM bundles. More follow the same pattern.

```typescript
const skillEntries = [
  "src/skills/setup.mts",
  "src/skills/mcp-setup.mts",
];

for (const entry of skillEntries) {
  const name = entry.replace("src/skills/", "").replace(".mts", "");
  await build({
    entryPoints: [entry],
    bundle: true,
    platform: "node",
    target: "node22",
    format: "esm",
    outdir: "dist/skills",
    outExtension: { ".js": ".mjs" },
    external: ["better-sqlite3"],
    sourcemap: true,
    minify: false,
  });
  console.log(`Built skill: dist/skills/${name}.mjs`);

```

## 4. npm Scripts

```json
{
  "scripts": {
    "build":      "node --import tsx esbuild.config.mts",
    "dev":        "node --import tsx --watch esbuild.config.mts",
    "test":       "vitest run",
    "test:watch": "vitest",
    "test:coverage": "vitest run --coverage",
    "lint":       "eslint src/",
    "format":     "prettier --write src/",
    "typecheck":  "tsc --noEmit",
    "sync-claude-plugin": "cp .github/plugin/plugin.json .claude-plugin/plugin.json",
    "prepublishOnly": "npm run build && npm run sync-claude-plugin"
  }
}
```

Note: `typecheck` runs `tsc --noEmit` for static type checking only. TypeScript is not compiled as part of the build pipeline — esbuild handles `.mts` files directly.

## 5. External Dependencies

The following packages are marked `external` in esbuild (not bundled):

| Package | Used by | Reason |
|---------|---------|--------|
| `better-sqlite3` | Hooks, MCP, CLI | Native SQLite bindings; must not be bundled |
| `@modelcontextprotocol/sdk` | `src/mcp/server.mts` | Resolved at runtime via npm |
| `simple-git` | Hooks, core | External npm package |
| `yaml` | Core, hooks | External npm package |

## 6. TypeScript Configuration Notes

The project uses `"type": "module"` in `package.json`. Source files use `.mts` extension. TypeScript is used for type checking (`tsc --noEmit`) and editor support, but esbuild handles the `.mts` → `.mjs` conversion directly via its built-in TypeScript loader — no separate `tsc` compilation step.

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "lib": ["ES2022"],
    "strict": true,
    "noImplicitAny": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "exactOptionalPropertyTypes": true,
    "noImplicitReturns": true,
    "noFallthroughCasesInSwitch": true,
    "skipLibCheck": true
  },
  "include": ["src/**/*", "esbuild.config.mts"],
  "exclude": ["node_modules", "dist", "tests"]
}
```

Required engine: `node >= 22.0.0` (esbuild `target: "node22"`).

## 7. Build Output Verification

After running `npm run build`:

```bash
# 1. Verify all 6 hook bundles exist
ls dist/hooks/
# Expected: delegation-enforcer.mjs  hud-emitter.mjs  keyword-detector.mjs
#           model-router.mjs  stop-continuation.mjs  token-tracker.mjs

# 2. Verify skill bundles
ls dist/skills/
# Expected: mcp-setup.mjs  setup.mjs

# 3. Verify MCP server bundle
ls dist/mcp/
# Expected: server.mjs  (and .mjs.map sourcemap)

# 4. Verify CLI bundle exists and is non-empty
ls -lh bin/omp.mjs

# 5. Verify no TypeScript type errors
npm run typecheck

# 6. Run tests
npm test
```
